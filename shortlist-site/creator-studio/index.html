Operating process

Use the same six phases every time.

Source: docs/creator-operating-process.md
01

Build the app

Ship the actual product journey before chasing aesthetics.

  • Working app with run instructions.
  • Core user journey implemented end to end.
  • Safe seed data and known privacy or claim risks.
02

Research UX patterns and options

Use the reskin skill to generate 10 creative directions before implementing a visual system.

  • Infer journeys from code, docs, schema, git history, and screenshots.
  • Create rough wireframes and score each direction.
  • Choose the top direction before spending production time.
03

Build the better UX pattern

Integrate the chosen visual and interaction model into the real app.

  • Responsive production UI.
  • Recording-friendly first viewport and state changes.
  • Demo data staged for video capture.
04

Finalize and publish

Make the app reliable, safe, and shareable before it becomes the video subject.

  • Lint and build pass.
  • Clean-browser demo path verified.
  • No private data, unsafe claims, or credentials visible.
05

Script the video

Turn the project into a proof-backed short-form story.

  • Hook variants, script, shot list, and screen-recording plan.
  • B-roll, captions, platform notes, and follow-up ideas.
  • Claim and privacy gate before filming.
06

Upload final cut for AI review

Review the exported video before publishing, then feed the learning back into the next build.

  • First-frame clarity, retention risk, pacing, audio, and captions.
  • 9:16 app-screen legibility and privacy safety.
  • Pass/fix decision plus post-publish metrics loop.

Feedback pipeline

The final cut should be reviewed before publishing.

First frame

Can a muted viewer understand the premise before scrolling?

Proof

Does the video show the app working, not just describe the app?

Legibility

Do app screens, captions, and overlays survive the 9:16 crop?

Pacing

Are cuts fast enough to hold attention while still preserving comprehension?

Claims

Are outcomes sourced, personal, or softened to accurate language?

Learning

Can the posted result teach the next UX direction, hook, or app idea?

How new things go here

Recommended route pattern

Keep the root as the internal index. Add each project, reskin readout, script package, or final-cut review as a separate route with its own allowlist.

  • /creator-studio/ for this operating hub.
  • /project-name-reskin/ for UX options.
  • /project-name-video-package/ for scripts and shot lists.
  • /project-name-review/ for final-cut AI review notes.
